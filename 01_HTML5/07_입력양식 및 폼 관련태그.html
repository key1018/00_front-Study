> post 방식 : 사용자가 입력한 값들이 url에 노출되지 않음, 전달값에 대한 크기 제한 없음

* text와 관련된 input태그의 타입

type="text"

- 한 줄짜리 텍스트를 입력할 수 있는 텍스트 상자(type속성 생략시 기본값)

type="password"

- 비밀번호 및 주민번호를 입력할 수 있는 텍스트 상자 (사용자가 입력한 값 노출X)

type="search/url/email/tel"

search : 사용자가 입력할 때 값을 초기화 할 수 있는 "X"버튼이 존재하는 태그

url : 사용자가 입력한 값에 http://가 삽입되어있는지 검사하는 태그 => http://가 없는 경우 "url을 입력해야합니다."라는 문구 출력됨

email : 사용자가 입력한 값에 @가 삽입되어있는지 검사하는 태그

tel : 숫자키 입력창 뜸, 모바일에서 확인 가능! 웹에서는 확인 불가능

* 라디오버튼과 체크박스 관련 input태그의 타입

라디오 버튼 (type="radio")

  1. 제시한 여러개의 값들 중에서 오로지 한 개만 선택받아야할 때
    (name 속성값이 똑같은 것들끼리 하나의 그룹으로 지어짐) => 그룹 내에서 오로지 한 개만 체크가능하게 됨
    => 그룹지정 안했을 경우 : 전체 선택 가능
    => name = "키값" 을 동일하게 작성하여 한 개의 그룹으로 묶기

  2. 한 번 체크하면 체크를 해지할 수 없음

  3. 제출시 현재 선택된(checked) 라디오버튼의 value값이 넘어감
    (즉, 해당 라디오버튼 선택시 어떤 값을 넘길껀지 value값을 명시해줘야됨! )

체크박스 (type="checkbox")

- 제시한 여러개의 값들 중에서 여러개 선택가능한 경우
- 제출시 현재 선택된(checked) 체크박스의 value값들이 넘거감
- 체크된 값들을 해지할 수 있음

첨부파일 :

type="hidden"

- 특정값을 사용자에게 직접 입력받지 않고, 화면에 노출시키지 않고, 근데 넘기고하는 값이 있는 경우 < input type="hidden" > 요소를 활용하여 숨겨놓은 뒤 값을 넘길 수 있다. 단, 입력할 당시 넘기고자하는 key값 + value값을 함께 작성해야 한다.



input type="submit | reset | button"

- submit : 입력값을 넘기고자 할 떄 사용 value="제출" (submit타입의 기본값)
- reset : 입력값을 초기화하고자 할 떄 사용 value="초기화" (reset타입의 기본값)
- button : 클릭만 가능 value값은 직접 명시해야함

button [type="submit | reset | buttion"]

- 문구가 자동으로 적용되지 않으므로 반드시 시작과 종료태그 사이에 문구를 작성해야함
- 기본값 : type="submit" 이므로 타입 명시는 필수이다
- CSS면에서 < input type="button" > 보다 <button> </button> 타입에서 더 다양한 스타일이 가능하다

* select태그와 option태그

- 사용자에게 직접 키보드로 값을 입력받는게 아니라 내가 제시한 여러개의 옵션(목록)들 중에서 선택할 수 있게끔 제공하고자 할 때
- 제출시 현재 선택된 option값이 넘어감
각각의 option에 value값을 명시하지 않았을 경우 content영역 (시작태그와 종료태그 사이)의 값이 넘어감(사용자에게 보여지는 값)
value값을 명시했을 경우 value값이 넘어감
=> value속성을 따로 명시해서 실제로 넘기는 값을 value값으로 지정할 수 있음(사용자에게 안보여짐)

HTML5에서 새로 추가된 기능 : dataList

- 사용자에게 직접 키보드로 값을 입력받을 수도 있고 목록에서 선택받을 수도 있음(멀티기능 가능)
