자바스크립트 개요

1. 자바스크립트

웹 브라우저에서 많이 사용되는 "인터프린터 방식"의 스크립트 언어
- 인터프린터 방식 : 코드를 한줄 한줄씩 읽어가면서 바로 실행하는 방식(실시간으로 텍스트를 읽어서 실행)   <-반대-> 컴파일 방식 => 컴파일 과정이 없기 때문에 코드에 문법적인 문제가 있다면 그 코드가 실행되는 시점에 오류를 알려줌 (오류파악이 어렵다)

2. 자바스크립트의 장단점

* 장점

- 컴파일 과정 없이 인터프리터로 코드를 한줄씩 읽어가며 바로 실행되기 때문에 "수행속도"가 빠르다
- 코드 작성이 간단해서 초보자가 접하기 쉬운 언어(단순한 구조, 원칙)
ex) 접근제한자의 개념없음, 변수 선언시 자료형 정해둘 필요없음, ..

* 단점

- 웹에 특화된 기술 => 내부에서 제공하는 기능이 제한적
- html내에 소스코드를 같이 작성하면 외부에 공개됨 => 보안에 취약점 발생

3. 자바스크립트 사용법

  1. inline(인라인) : 요소내에 간단한 소스코드를 작성해서 실행되게 하는 방법
  2. internal(내부방식) : html문서내에 script태그를 작성하고 그 안에 소스코드를 작성해서 실행되게 하는 방법
  3. external(외부방식) : 별도의 .js 파일로 소스코드를 작성해두고 가져다가 실행되게 하는 방법

1) inline(인라인) 방식

태그 내에 간단한 소스코드를 작성해서 실해되게하는 방법
< 태그 on이벤트명="해당 요소에 해당이벤트 발생시 실행시킬 소스코드" >

2) internal 방식

현재 html문서내에 script태그 안에 소스코드 작성
script태그는 head태그든, body 태그든 어디서든 다 작성가능

3) external (외부방식)

별도의 .js 파일로 소스코드를 작성해두고 가져다 사용하는 방법
